对动态代理的理解

动态代理就好像中介公司一样，我们买什么东西，找中介，我们也不关心具体是谁去买的。

静态代理不一样，我需要明确指定是谁做为中间者。

动态代理的理解

动态代理实际上是一种动态注入方法的方法。

比如 Retrofit，Retrofit 自身不可能知道我们在使用过程中，到底有哪些网络请求。这些网络请求的定义，是使用 retrofit 的人，根据业务需求自己定义的请求方法。

retrofit 提供了定义这些请求方法的规范，我们可以按照这些规范，做好方法的定义（接口的定义），不需要我们提供这些方法的实现，retrofit 就可以使用动态代理的方式，去实现这些方法。

动态代理相对于 定义接口，让开发人员提供具体实现的这种方式 来说，是一种反向的操作，动态代理提供了方法的实现，由开发人员再去定义所需要的接口。

Compose 快速上手

用 Compose 写出一个简单界面

Compose 一些基本概念问题

  1. Compose 独立于平台
    1. 不依赖于 android 更新，可以独立发布: ConstraintLayout, AppCompat, ViewModel, RecyclerView 等
    2. Compose 更激进，独立于 Android，与 Android 无关，可以跨平台
    3. 独立于平台的一个最明显的用处就是在 Android Studio 中可以进行更强大的预览功能
    4. 目前 Compose 支持 PC 已经进入 beta 版、也正在推进 Web 端
  2. 声明式 UI
    1. 只需要对界面进行一次性的声明，而不需要在不同条件下对界面进行操作。
