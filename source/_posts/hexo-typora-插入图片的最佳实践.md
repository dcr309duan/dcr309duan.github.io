# 背景

博客中插入图片等资源是不可避免的，常见的在博客中插入图片，有以下两种方案：

1. 将图片上传到图床中
2. 将图片内置在网站中

## 图床方案

使用图床方案有以下优点：

1. 图片链接固定，我们在 markdown 中直接使用 `![](https://xxx.png)` 的方式就可以，不需要额外的配置
2. 图片的访问独立于我们的网站，如果网站的访问量很大，图片流量单独打到图床上，不会对我们的网站服务器造成压力

但是图床方案最大的问题在于成本：

1. 图床成本：图床可以使用三方免费图床或者 oss 来实现，使用 oss 是一笔额外的成本，而免费图床可能不稳定
2. 维护成本：图床迁移，我们的图片链接就会变，需要将博客中的图片链接全部替换

如果只是个人博客，我们网站的访问量不会很大，所以再单独去设置一个图床有点大材小用了。

## 网站内置图片

hexo 支持将图片内置在网站中，配置也并不复杂，对于个人博客来说，内置网站维护成本最低（应付那点访问量足够了）

hexo 支持配置资源文件夹，参考文档：https://hexo.io/zh-cn/docs/asset-folders

# 目标

1. 发布到网站上能够正常显示图片
2. 搭配 typora 使用，在 typora 中能够正确显示图片
3. 图片分文件夹存储，方便管理
4. 在文章中插入图片时，尽可能自动化

# 配置和踩坑记录

## hexo 配置

hexo 配置文档：https://hexo.io/zh-cn/docs/asset-folders

在 hexo 的配置文件（`_config.yml`）中，需要打开如下几个选项：

```yaml
post_asset_folder: true
marked:
  prependRoot: true
  postAsset: true
```

现在，我们实现的效果如下：

使用 `hexo new paper` 时，自动生成对应的资源文件夹

我们可以通过 `![](image.png)` 的方式插入图片，只要 `image.png` 位于文章对应的资源目录下。

## typora 配置

现在我们需要将放置图片 + 引用图片的过程尽可能的自动化。

### 自动将图片 copy 到资源文件夹

这一点我们可以在 typora 设置中完成：

![image-20240505235242256](image-20240505235242256.png)

这样，当我们将一张图片粘贴或者拖动到 typora 中时，typora 会自动将图片 copy 到对应的目录下。

但是仍然存在一个问题，就是文章中，引用的图片路径是按照相对路径的方式，例如：

```markdown
![](./paper_name/xxxx.png)
```

而我们期望的方式是：

```markdown
![](xxx.png)
```

### 设置 typora 的索引根路径

如果我们去掉图片的相对路径，那么 typora 就会找不到这张图片，无法正常在 typora 中进行渲染。

幸运的是，typora 支持设置图片索引的根路径，只需要这样即可：

```markdown
---
typora-root-url: ./paper_name
---
```

这一点可以通过 hexo 的模板来实现，例如，我们可以配置如下的模板（例如：`scaffolds/post.md`）：

```
---
title: {{ title }}
date: {{ date }}
tags:
typora-root-url: ./{{ title }}
---
```

这样，我们使用 `hexo new` 生成文章时，就会自动带上这个属性

# 遗留问题

## 文章标题存在空格的处理

如果我们的标题存在空格，那么 `{{ title }}` 的方式会保留空格，而实际的文件名会将空格替换为 `-`。

例如：如果文章名为：`test 1`，那么实际的文件名会为 `test-1`

这一点暂时还没找到好的解决方案，需要手动改一下

## 添加图片后，包含 `/`

typora 经过上述的配置，在粘贴图片后，生成的图片路径为：

```markdown
/xxx.png
```

而我们期望的是：

```markdown
xxx.png
```

我们不需要这个 `/`，否则 hexo 中无法正确找到图片。这一点暂时也没找到好的解决方案，需要手动删除一下。
